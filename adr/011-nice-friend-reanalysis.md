# ADR-011: Codex (nice-friend) による反復的再分析

## Status

Accepted

## Context

自律エージェントによる分析は「自分で作った分析を自分でレビューする」バイアスがかかりやすい。見落とし視点や代替解釈が欠落するリスクがある。

## Decision

**OpenAI Codex CLI (nice-friend skill) をセカンドオピニオンとして活用する。**設計判断や分析結果について、Codex に具体的な質問を投げて反論・補完意見を得る。有益なフィードバックは分析に反映する。

## Alternatives Considered

- **Claude 同士のピアレビュー**: 同一モデルファミリーのため視点の多様性が限定的。
- **人間レビューのみ**: 品質は高いが、プロジェクトオーナーの時間制約上、全分析のレビューは困難。
- **自動テストのみ**: 数値検証は可能だが、解釈の妥当性や見落とし視点の発見は困難。

## Assumptions

- 異なる AI モデル（Claude vs Codex）は異なるバイアスを持ち、相互補完的に機能する
- Codex の物理学知識は orbital mechanics の考察に十分な水準
- nice-friend skill の呼び出しオーバーヘッド（~30-60秒）は分析品質向上に見合う

## Consequences

- 単一 AI の視点に閉じない多角的な分析が可能
- Codex の指摘により発見された例：座標系の明確化（EP02）、ノズル熱サイクル損傷（EP05）、マージン連鎖分析（クロスエピソード）
- 追加の API コスト（Codex 側）が発生するが、分析品質の向上に見合う
